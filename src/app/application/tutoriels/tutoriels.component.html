Mini tutoriel sur Angular

C'est quoi? : Un framework front-end web open source qui permet de créer des applications web multiplateformes. Cela nous permet de faire des pages SPA (c'est-à-dire Single Page Application) ou SSO (Single Sign On). De plus, cela nous permet de faire l'injection de dépendance, de créer des composantes pour cacher l'implémentation et de garder la simplicité du développement web.

Pour installer Angular globalement sur les machines, il suffit d'éxécuter la commande suivante sur un terminal: npm install -g @angular/cli@10.0.0

Suite à l'installation, il faut également installer et configurer le serveur Json:

  • npm i g json-server
  • npm i faker

Installation optionnelle du compilateur Typescript pour travailler depuis un projet Angular:

  • npm install -g typescript@latest
  • npm install -g ts-node

Pour utiliser les commandes d'Angular, il suffit d'utiliser les commandes qui commencent par "ng" suivi de ce qu'on veut faire, voici quelques exemples:

Générer un nouveau projet:
ng new my-app
Générer un nouveau composant:
ng g c my-component
Générer une nouvelle classe:
ng g cl my-class-model
Générer une nouvelle interface:
ng g i my-interface
Générer des services:
ng g s my-service

Mini tutoriel CRUD

Qu'est-ce que le CRUD? : Ce sont les méthodes qui permettent de communiquer avec ou de sauvegarder des données avec la base de données.

  1. [C]reate : Permet de créer un enregistrement de l'objet courant et de l'insérer dans une base de données.
      2. Ex: Création d'un employé dans une banque.
  2. [R]ead : Permet de lire/afficher les données de la base de données.
      3. Ex: Lire/Afficher une liste d'employés.
  3. [U]pdate : Permet de faire des mises à jour sur les enregistrements, c'est-à-dire pouvoir les modifier.
      4. Ex: Mettre à jour le salaire d'un employé.
  4. [D]elete : Permet de supprimer un ou plusieurs enregistrements d'une table dans la base de données.
      5. Ex: Supprimer un employé d'une banque.

Pourquoi est-ce nécessaire? : Sans le CRUD, il n'y aurait pas d'autres moyens pour sauvegarder des enregistrements importants. Ce sont les opérations de base pour pouvoir stocker les informations dans une base de données et faire la persistance de données.

Les méthodes du CRUD sont semblables aux requêtes LMD(Language de Modification de Données) en SQL. Elles ont les mêmes fonctionalités, mais écrites différemment :

CRUD SQL POSTMAN
Create Insert Post
Read Select Get
Update Update Put
Delete Delete Delete

Mini tutorial sur les relations

Qu'est-ce que c'est? : Les relations représentent les liens des objets qui sont reliés, c'est-à-dire leurs rôles et ce qu'ils peuvent affecter.

Il existe plusieurs type de relations qu'un objet peut posséder, il sont montrés dans l'image ci-dessous.

L'association : Le premier objet A utilise le deuxième objet B ou bien vice-versa.

L'héritage : Un objet hérite d'un autre objet. Ex : Objet A hérite de Objet B.

La dépendance : Deux objets ou plus sont dépendants entre eux. S'il y a un changement dans un des objets, les autres objets vont être affectés également.

Composition : Un objet ne peut pas exister sans l'autre, en d'autres termes, on ne peut pas supprimer l'objet parent si un objet enfant existe. Ex : un livre ne peut pas exister sans une bibliothèque. Si la bibliothèque est détruite, le livre qui fait partie de cette bibliothèque est également détruite.

Aggrégation : C'est comme la composition, mais à un niveau plus faible. C'est-à-dire que les objets peuvent exister de facon indépendante. Ex : une banque et des employés. Si on enlève la banque, les employés existent encore.

Parmi les relations, il existe également 3 types d'associations possibles :

  1. One-To-One : L'objet A est en relation de 1 à 1 avec l'objet B. C'est-à-dire que l'objet A devient un attribut dans l'objet B ou vice-versa.
  2. One-To-Many : L'objet A est en relation de 1 à plusieurs avec l'objet B. Ce qui veut dire que dans l'objet A, il y aura un tableau d'objet B.
  3. Many-To-Many : L'objet A est relation plusieurs à plusieurs avec l'objet B. Il faut créer un tableau intermédiaire qui relie les deux objets ayant cette relation. Cela permet de contenir les attributs primaires des deux objets.

ATTENTION : Il faut savoir que le nombre de classes dans un schéma relationnel doit être supérieur ou égal au nombre de classes dans le diagramme UML. Le seul cas possible est lors d'une relation many-to-many, ce qui veut dire qu'il faut créer la classe intermédiaire.
