Mini tutoriel sur Angular

Qu'est-ce qu'Angular? : C'est un framework open source (License gratuite) front-end qui permet de créer des applications web multiplateformes. Cela nous permet également de faire des pages SPA (c'est-à-dire Single Page Application) ou SSO (Single Sign On). De plus, Angular nous permet de faire l'injection de dépendance (garder les composants légés et délaisser la logique et les données aux services), de créer des composantes pour cacher l'implémentation et de garder la simplicité du développement web.

Tout d'abord, installer le terminal Git en suivant ce lien afin d'exécuter les commandes suivantes.

Ensuite, installer Node.JS sur les machines en prenant la version LTS afin de pouvoir exécuter les commandes CLI d'Angular : Lien de téléchargement de Node.JS

C'est quoi Node? : C'est une librairie qui contient toutes les dépendances que NPM (Node Package Manager) peut exécuter. NPM est un gestionnaire qui permet de gérer les dépendances, ces dépendances se retrouvent généralement dans le fichier "package.json" du projet Angular.

Ensuite, il faut installer Angular globalement sur les machines, il suffit d'éxécuter la commande suivante sur un terminal : npm install -g @angular/cli@10.0.0

Suite à l'installation, il faut également installer et configurer le serveur Json :

  • npm i g json-server
  • npm i faker

Installation optionnelle du compilateur Typescript pour travailler depuis un projet Angular :

  • npm install -g typescript@latest
  • npm install -g ts-node

Pour utiliser les commandes d'Angular, il suffit d'utiliser les commandes qui commencent par "ng" suivi de ce qu'on veut faire, voici quelques exemples :

Générer un nouveau projet:
ng new my-app
Générer un nouveau composant:
ng g c my-component
Générer une nouvelle classe:
ng g cl my-class-model
Générer une nouvelle interface:
ng g i my-interface
Générer des services:
ng g s my-service

Pour en connaitre davantage pourquoi on utilise Angular, cliquez sur ce lien.

Mini tutoriel CRUD

Qu'est-ce que le CRUD? : Ce sont les méthodes qui permettent de communiquer avec une base de données ou de les manipuler.

  1. [C]reate : Permet de créer un enregistrement d'un objet et de pouvoir l'insérer dans une base de données.
      2. Ex: Création d'un employé dans une banque.
  2. [R]ead : Permet de lire/afficher les données/objets d'une base de données.
      3. Ex: Lire/Afficher une liste d'employés.
  3. [U]pdate : Permet de faire des mises à jour sur les enregistrements, c'est-à-dire pouvoir les modifier.
      4. Ex: Mettre à jour le salaire d'un employé.
  4. [D]elete : Permet de supprimer un ou plusieurs enregistrements d'une table dans la base de données.
      5. Ex: Supprimer un employé d'une banque.

Pourquoi est-ce nécessaire? : Sans le CRUD, il n'y aurait pas d'autres moyens pour sauvegarder des enregistrements importants. Ce sont les opérations de base pour pouvoir stocker les informations dans une base de données et faire la persistance de données.

Les méthodes du CRUD sont semblables aux requêtes LMD(Language de Modification de Données) en SQL. Elles ont les mêmes fonctionalités, mais écrites différemment :

CRUD SQL POSTMAN
Create Insert Post
Read Select Get
Update Update Put
Delete Delete Delete

Mini tutorial sur les relations d'objets

Qu'est-ce que c'est? : Les relations représentent les liens des objets qui sont reliés ensemble, c'est-à-dire leurs rôles et comment ces rôles intéragissent avec d'autre objets.

Il existe plusieurs types de relations qu'un objet peut posséder, ils sont montrés dans l'image ci-dessous.

Remarque : Les types de relations dans la photo sont représentés de façon unidirectionnel

L'association : La première classe A utilise la deuxième classe B ou bien vice-versa.
Parmi les relations, il existe également 3 types d'associations possibles :

  1. One-To-One : La classe A est en relation de 1 à 1 avec la classe B. C'est-à-dire que la classe A devient un attribut dans la classe B ou vice-versa.
  2. One-To-Many : La classe A est en relation de 1 à plusieurs avec la classe B. Ce qui veut dire que dans la classe A, il y aura un tableau de la classe B.
  3. Many-To-Many : La classe A est en relation plusieurs à plusieurs avec la classe B. Il faut créer un tableau intermédiaire qui relie les deux classes ayant cette relation. Cela permet de contenir les attributs primaires des deux classes.

L'héritage/généralisation : Une classe hérite d'une autre classe.
Ex : Classe A hérite de la classe B, car B est la classe parent

La réalisation/implémentation : La classe A implémente la classe B, car B est une interface

La dépendance : La classe A dépend sur une autre classe B. S'il y a un changement dans la classe B, alors il faudra également apporter des modifications à la classe A.

Composition : Une classe ne peut pas exister sans l'autre, en d'autres termes, si on supprime la classe A qui se fait composer par la classe B, alors la classe B ne va plus exister.
Ex : Un livre ne peut pas exister sans une bibliothèque. Si la bibliothèque est détruite, le livre qui fait partie de cette bibliothèque est également détruite.

Aggrégation : C'est comme la composition, mais à un niveau plus faible. C'est-à-dire que les classes peuvent exister de facon indépendante.
Ex : Une banque et des employés. Si on enlève la banque, les employés existent encore.

ATTENTION : Il faut savoir que le nombre de classes dans un schéma relationnel doit être supérieur ou égal au nombre de classes dans le diagramme UML. Le seul cas possible est lors d'une relation many-to-many, il faut donc créer la classe intermédiaire.
